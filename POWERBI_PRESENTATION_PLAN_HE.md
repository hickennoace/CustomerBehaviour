## 3. מסנני פרוסות — בדיוק מה הולך לאן

הכלל: **שים מסנן פרוסה בעמוד רק אם הוא משנה באופן גלוי ≥ 3 ויזואלים באותו עמוד.** כל השאר עובר לחלונית מסנן מוסתרת.

### 3.1 מסנני פרוסה ברמת עמוד (גלויים, פינה עליונה ימנית בכל עמוד)

| עמוד                     | מסנן פרוסה                | שדה                                          | סגנון              |
|--------------------------|---------------------------|----------------------------------------------|--------------------|
| **סיכום מנהלים**         | טווח תאריכים              | `dim_players[last_purchase_date]`            | מחוון "בין"        |
| **סיכום מנהלים**         | דרגה                      | `dim_players[rank]`                          | אריח / אופקי       |
| **פסיכולוגיית שחקנים**  | דרגה                      | `dim_players[rank]`                          | אריח / אופקי       |
| **פסיכולוגיית שחקנים**  | סטטוס משלם                | `dim_players[is_payer]`                      | מתג (2 כפתורים)    |
| **פסיכולוגיית שחקנים**  | דלי זמן משחק *(ראה §3.3)*| עמודה מחושבת                                 | רשימה נפתחת        |
| **הכנסה אבודה**          | דרגה                      | `dim_players[rank]`                          | אריח / אופקי       |
| **הכנסה אבודה**          | קטגוריה                   | `fact_abandoned_carts[category]`             | רשימה נפתחת (רב)  |
| **הכנסה אבודה**          | דלי ערך עגלה *(§3.3)*    | עמודה מחושבת                                 | רשימה נפתחת        |

### 3.2 סנכרון מסנני פרוסה בין עמודים
עבור **דרגה**: תצוגה → סנכרן מסנני פרוסה → סמן את שלושת העמודים כדי שהקהל יראה שהמצב נשמר בין עמודים. **אל** תסנכרן את טווח התאריכים — עמוד 2 (פסיכולוגיה) צריך להציג תמיד את ההפצה המלאה; סינון תאריכים שלו יוצר רעש בחישובי מתאם.

### 3.3 שתי עמודות מחושבות להוסיף ל-`dim_players`

```DAX
Playtime Bucket =
SWITCH (
    TRUE (),
    dim_players[total_playtime_hours] < 10,   "0–10 ש' (תייר)",
    dim_players[total_playtime_hours] < 50,   "10–50 ש' (קז'ואל)",
    dim_players[total_playtime_hours] < 200,  "50–200 ש' (רגיל)",
    dim_players[total_playtime_hours] < 500,  "200–500 ש' (ליבה)",
    "500 ש'+ (הארדקור)"
)
```

```DAX
Cart Value Bucket =
SWITCH (
    TRUE (),
    dim_players[cart_total_value] = 0,    "ריק",
    dim_players[cart_total_value] < 10,   "מתחת ל-$10",
    dim_players[cart_total_value] < 25,   "$10–25",
    dim_players[cart_total_value] < 50,   "$25–50",
    "$50+"
)
```

הגדר את *מיין לפי עמודה* שלהם לעמודת מספר שלם מוסתרת (`Playtime Bucket Sort`, `Cart Value Bucket Sort`) כדי שהרשימה הנפתחת לא תהיה בסדר אלפביתי.

### 3.4 חלונית מסנן מוסתרת (מסננים מתקדמים, כבויים כברירת מחדל בזמן מצגת)
- `dim_players[first_join_date]` — לקידוח קבוצות בשאלות ותשובות
- `fact_purchases[item]` — כאשר מנהל שואל "מה עם Mythical ספציפית?"
- `dim_players[total_transactions] > 0` — מתג מהיר "בסיס משלמים בלבד"

### 3.5 התנהגות סינון צולב
- הגדר *ערוך אינטראקציות* של כל גרף כך שמסנני פרוסה **מסננים**, אך גרפי עמודות/קו רק **מדגישים** זה את זה (לא סינון מלא). הדגשה שומרת על הקשר; סינון מלא בזמן הדגמה חיה גורם לגרפים ריקים ולמציג קפוא.
- טבלת הלווייתנים המובילים בעמוד 1 צריכה להיות מוגדרת **ללא אינטראקציה** מגרף ההכנסות לפי קטגוריה — הטבלה היא "רשימת מי", לא "מי בקטגוריה הזו".

## 4. כוריאוגרפיה של המצגת (תסריט להדגמה חיה)

1. **פתח בעמוד 1.** התחל עם כרטיס **סך הכנסות** ← עבור מיד ל-**% נטישת עגלה** — בסס שמספר הכותרת מסתיר בעיה.
2. **השתמש במסנן הדרגה**, לחץ `Default` ← ARPPU קורס, הנטישה מזנקת. זה הרגע שמוכר את העמוד הבא.
3. **עבור לעמוד 2.** הצג את מפת החום: "זמן משחק בקושי מתאם להוצאה (r ≈ 0.1) — מעורבות אינה המנוף." שלוף את מתג המשלם להשוואה.
4. **עבור לעמוד 3.** הצג את משפך הגיבור. נחת על כרטיס "הכנסה ניתנת לשחזור" כקריאה לפעולה ("אם נשחזר 20% מזה, זה $X ברבעון הבא").
5. **חזור לעמוד 1** עם הדרגה עדיין מסוננת ל-`Default` לסגור את הלולאה ויזואלית.
